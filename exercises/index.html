1: Warm Up

Prime the ol' brain-pump with some basic HTML/CSS.

  1. See the link to the exercise page above? Go ahead and open the exercise page up in Chrome. Behold the beauty.
  2. Now, open the page up in your text editor. We're going to make it even more magnificent.
  3. In general, the important styles for the exercise are in a <style> tag in the <head> of the page. Do you see it?
  4. Ok, let's edit. Change the color of the h1 to make it more subtle. Try #333333 if you like. Save the file, then switch back to Chrome and reload the page to view the changes.
  5. The paragraph text is too small. Back to your editor again to increase the font-size. Try 12px or 14px. (Or 13px if you aren't superstitious). Again, save the changes, switch back the Chrome and reload the page to view the changes. How is it lookin'?
  6. Now we'll spruce up the layout a little. Let's make this a "fixed-width" layout by setting the body width to 800px. Save and refresh.
  7. Stuck on anything? You can always take a look at the completed exercise from the other link above.

Already twiddling your thumbs? Try these extra credit warm-ups.

  1. Add another paragraph and give it a class of your choosing. Then, add style rules for this class that center the text, add a border, a little padding, and a tasteful background color.
  2. Add an <aside> tag after the <main> with a few paragraphs in it. Using CSS floats, make <main> and <aside> side-by-side columns.

2: A Basic Layout

Start by examining a simple fixed layout and then make it fluid using percentage-based widths.

  1. First, open up the page in Chrome and examine it with the dev tools, using the Elements pane with the Styles tab to see what is happening. Why is <main< next to <aside?
  2. Turn on Device mode. Pick a few devices and see what happens to the layout. Each time you change the device, you may need to reload the page. (If you can't fit the device screen on your laptop's screen, enable the "fit" checkbox below the device model dropdown.)
  3. Using the dev tools, find where the <main<'s width is set. Edit the width in-place.
  4. Reload the page to obliterate the temporary changes you made in the dev tools and return to viewing the saved CSS.
  5. Note the fat, round cursor - the better to simulate fat fingers!
  6. The simplest way to make a layout more responsive is to set all your widths in percentages. Using the dev tools, change the widths of <main< and <aside< to 70% and 30%, respectively.
  7. Whoops! Something ain't right. Turns out, in addition to width, we have some margin infecting our layout. Let's go ahead and make the margin fluid as well, and subtract it from the width of <main<. In dev tools, make it's width: 68% and margin-right: 2%.
  8. Now, go through a few devices (or drag the viewport handles) and see how the layout behaves. Turn off device-mode to see how it looks full-width.
  9. Congratulate yourself for making a responsive layout! Ok, maybe it's responsive "lite". Baby steps!

Extra Credit

  1. Add a third column before or after. Add styles to make them all fit nicely with percentage-based widths!
  2. Do you know about absolute positioning (TODO: link to tutorial or MDN)? If you do, try removing the float: left from <aside< and using absolute positioning to place it to the right of <main<.

3: Baby's First Media Query

Add a simple media query to an otherwise unassuming fluid layout.

  1. Open up the exercise page in Chrome - should look familiar! Turn on device-mode and pick a device. This layout needs some love at narrower screens!
  2. Drag the viewport width handle around. Around what width does it look too skinny? This is the "breakpoint" we want to target for some different styling.
  3. Open the page up in your editor. Below the other styles, add a media query block that targets screens that are as wide as or narrower than the breakpoint you found above. It should look like @media screen and (max-width: BREAKPOINT_IN_PX){ ... }.
  4. Any styles within this block will only be applied to narrower screens. To see the effect, add a style rule within the media query block that changes the background color of <main< to "purple". Save the changes in your editor, reload the page in Chrome and drag the viewport width around. Isn't it pretty?
  5. Nice! That was utterly useless. What we'd like to do is change the layout for those narrower screens. Let's make <main< and <aside< full-width instead of columns. In your editor, within your new media query, set their widths to "100%". Save, reload the page and behold!

6: Adjustable grids

Take a simple, crude grid layout and make it conform smoothly to the screen width.

  1. The starting exercise page is already responsive, but in a kind of crude way. In Chrome's device-mode, check out the page at various widths and note how the spacing on the right side of the grid behaves.
  2. First off, in dev tools, figure out which style is controlling the width of the grid items. Verify you have the right style by changing the value and seeing the impact.
  3. Let's make the grid fit it's container by using percentage-based widths. We want to choose a width that is cleanly divisible into 100 so there is no gap on the right side. Let's start with 25%. Just make this change in Chrome for now.
  4. Goof around with the device width and see what happens. What happens on narrower and wider screens?
  5. Four items per column is too many for our narrowest screens. Let's make our mobile-first default a 2-item-per-row grid. Go ahead and switch to your editor and make 50% the default width, save and reload.
  6. We really want this grid to work well at any width, so we are going to add some fine-grained breakpoints. Add a breakpoint for screens wider than 600px that switches to a 3-column layout. (Since 100/3 is 33.333333..., we will just use 33.333%)
  7. Add media queries for 4-column and 6-column layouts at screen widths that make sense to you. Check your work in Chrome and go back and adjust the breakpoints until the transitions between layouts feel right.

Extra Credit

  1. Make minor adjustments to the font-size and grid item height at each breakpoint. The general rule is, as screens get larger, noses get further away, so adjust slightly up at each breakpoint!
  2. Often, layouts feel out-of-proportion when they get too wide. Give our <main> a max-width of 1200px. Center it as well by adding margin: 0 auto.
  3. Difficulty: ♦♦. Give the first item in each row a special background-color. Use the nth-child selector (background). Try to get the first-item-in-row highlight working at all the breakpoints.
